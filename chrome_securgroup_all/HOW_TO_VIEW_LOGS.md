# 🔍 如何查看插件日志 - 完整指南

## 📍 重要概念

Chrome 插件的日志会输出到**不同的控制台**，取决于代码在哪里运行：

| 代码文件 | 运行位置 | 日志显示在 |
|---------|---------|-----------|
| **content.js** | 网页中 | **网页的控制台**（按F12） |
| **popup.js** | 弹出窗口中 | **弹出窗口的控制台**（右键→检查） |
| **background.js** | 后台 | **扩展服务的控制台** |

---

## ✅ 查看 Content Script 日志（最重要！）

这是你要看的主要日志，因为 content.js 负责操作页面上的复选框。

### 步骤：

1. **打开你的目标页面**（Maximo 页面或测试页面）

2. **按 F12 打开开发者工具**
   - Windows: `F12` 或 `Ctrl + Shift + I`
   - Mac: `Cmd + Option + I`

3. **确保选中 "Console"（控制台）标签**
   - 如果看到多个标签（Elements, Console, Sources等），点击 "Console"

4. **刷新页面**（按 F5）

5. **查找日志**
   ```
   [Checkbox Helper] 内容脚本已加载
   ```

### 截图说明：

```
┌─────────────────────────────────────┐
│  Chrome 浏览器                       │
│  ┌─────────────────────────────┐   │
│  │  你的网页内容                │   │
│  │  - 复选框                   │   │
│  │  - 其他内容                 │   │

│  ┌─────────────────────────────┐   │

│  │  已加载                      │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

## ✅ 查看 Popup 日志（可选）

如果你想调试 popup.js（弹出窗口的逻辑）：

### 步骤：

1. **点击插件图标**打开弹出窗口

2. **在弹出窗口上右键点击**

3. **选择 "检查"** 或 "Inspect"

4. **这会打开一个新的开发者工具窗口**

5. **切换到 Console 标签**

6. **点击弹出窗口中的按钮**，查看日志

## ❓ 常见问题

### 问题1：控制台完全没有输出

**可能原因：**
- 插件没有正确加载
- 内容脚本没有注入

**解决方法：**
1. 去 `chrome://extensions/`
2. 确认插件是启用状态
3. 点击刷新按钮 🔄
4. 刷新目标页面（F5）
5. 再次查看控制台

### 问题2：看到其他日志，但没有 "[Checkbox Helper]"

**可能原因：**
- 插件版本不是最新的
- manifest.json 配置有问题

**解决方法：**
1. 确认插件版本号是 1.0.3
2. 重新加载插件
3. 完全关闭浏览器后重新打开
