# 🧪 测试日志输出 - v1.0.4

## ✅ 已添加的日志

### 1. Content Script 启动日志
当页面加载时，会在**网页控制台**（F12）看到：
```
========================================
[CONTENT] ✅ 内容脚本已加载！
[CONTENT] 页面 URL: https://...
[CONTENT] 时间: 2026/5/12 17:xx:xx
========================================
```

### 2. Popup 启动日志
当你点击插件图标打开弹出窗口时，会在**弹出窗口的控制台**看到：
```
========================================
[POPUP] 弹出窗口已加载！
[POPUP] 时间: 2026/5/12 17:xx:xx
========================================
[POPUP] DOM元素已获取
[POPUP] 正在绑定按钮事件...
[POPUP] ✅ 所有按钮事件已绑定
```

### 3. 按钮点击日志
当你点击任何按钮时，会看到：
```
========================================
[POPUP] 🖱️ 用户点击了【全选】按钮！
[POPUP] 时间: 2026/5/12 17:xx:xx
========================================
```

## 📋 测试步骤

### 第1步：重新加载插件 ⚠️ 必须！

1. 打开 `chrome://extensions/`
2. 找到"批量复选框勾选助手"
3. **点击刷新按钮** 🔄
4. 确认版本号是 **1.0.4**

### 第2步：刷新目标页面

在你的 Maximo 页面上按 **F5** 刷新

### 第3步：打开控制台查看 Content 日志

1. 在 Maximo 页面上按 **F12**
2. 切换到 **Console** 标签
3. 应该立即看到：
   ```
   ========================================
   [CONTENT] ✅ 内容脚本已加载！
   [CONTENT] 页面 URL: ...
   [CONTENT] 时间: ...
   ========================================
   ```

### 第4步：打开 Popup 查看 Popup 日志

1. **点击插件图标**打开弹出窗口
2. **在弹出窗口上右键点击**
3. 选择 **"检查"** 或 "Inspect"
4. 这会打开一个新的开发者工具窗口
5. 切换到 **Console** 标签
6. 应该看到：
   ```
   ========================================
   [POPUP] 弹出窗口已加载！
   [POPUP] 时间: ...
   ========================================
   [POPUP] DOM元素已获取
   [POPUP] 正在绑定按钮事件...
   [POPUP] ✅ 所有按钮事件已绑定
   ```

### 第5步：点击按钮测试

1. 回到弹出窗口
2. 点击 **"全选"** 按钮
3. 在弹出窗口的控制台中应该看到：
   ```
   ========================================
   [POPUP] 🖱️ 用户点击了【全选】按钮！
   [POPUP] 时间: ...
   ========================================
   ```

4. 同时在**网页的控制台**中应该看到：
   ```
   [Checkbox Helper] 收到消息: {action: 'operateCheckboxes', ...}
   [Checkbox Helper] 操作复选框 - 选择器: input.cds--checkbox 操作: select
   [Checkbox Helper] 找到复选框数量: 13
   ...
   ```

## ❓ 如果看不到日志

### 情况1：网页控制台没有任何输出

**可能原因：**
- 插件没有正确加载
- 内容脚本没有注入

**解决方法：**
1. 去 `chrome://extensions/` 确认插件启用
2. 点击刷新按钮 🔄
3. 完全关闭浏览器后重新打开
4. 再次刷新页面

### 情况2：弹出窗口控制台没有输出

**可能原因：**
- 没有在弹出窗口上右键检查
- 弹出窗口已经关闭

**解决方法：**
1. 点击插件图标打开弹出窗口
2. **保持弹出窗口打开**
3. 在弹出窗口上右键 → 检查
4. 然后再点击按钮

### 情况3：看到日志但不是我们添加的格式

**可能原因：**
- 插件版本不是 1.0.4

**解决方法：**
1. 确认版本号是 1.0.4
2. 重新加载插件
3. 刷新页面
